# CoSensei Web - Security Risk Analysis Dashboard

A React-based web interface for displaying security risk analysis, code vulnerabilities, and security fixes generated by the CoSensei system.

## Features

- **Real Code Vulnerabilities**: Shows actual unsafe code with embedded security risks
- **Before/After Comparison**: Displays vulnerable code alongside secure versions
- **Detailed Risk Analysis**: CRITICAL, HIGH, and MEDIUM severity categorization
- **Security Fixes**: Specific recommendations for fixing each vulnerability
- **Multiple File Analysis**: Analyze different components (main.py, config.py, routes.py, models.py)
- **Dark Theme**: Modern, easy-on-the-eyes security dashboard UI

## Setup

1. Install dependencies:
```bash
npm install
```

2. Start the development server:
```bash
npm run dev
```

The app will be available at `http://localhost:3000`

## Building for Production

```bash
npm run build
npm run preview
```

## Integration with Python Backend

The React app expects a backend API at `http://localhost:5000/api/risks` that returns risk analysis data.

### Backend API Endpoint

`GET /api/risks`

Expected response format:
```json
{
  "summary": {
    "critical": 10,
    "high": 9,
    "medium": 3
  },
  "files": {
    "main.py": {
      "title": "main.py - Flask Application",
      "description": "...",
      "risks": [
        {
          "severity": "CRITICAL",
          "title": "Debug mode enabled",
          "description": "...",
          "line": "..."
        }
      ],
      "unsafeCode": "...",
      "secureCode": "...",
      "fixes": ["...", "..."]
    }
  }
}
```

## Fallback Demo Data

If the backend is not available, the app loads demo security analysis data showing:
- 4 example files: main.py, config.py, routes.py, models.py
- 22+ real security vulnerabilities
- Before/after code comparisons
- Specific fixes for each vulnerability

## Technologies Used

- **React 18**: UI framework
- **Vite**: Build tool and dev server
- **Axios**: HTTP client for API calls
- **CSS**: Custom styling with dark theme gradient background

## Features Highlights

### Risk Card Component
Displays:
- Severity badges (CRITICAL in red, HIGH in orange, MEDIUM in yellow)
- Vulnerability title and description
- Line of code showing the issue

### Code Comparison Component
Shows:
- Unsafe code with red border
- Secure code with green border
- Line-by-line fixes and improvements
- Toggle between unsafe and secure code views

### Security Summary Component
Displays:
- Total CRITICAL issues (must fix)
- Total HIGH risk issues (should fix) 
- Total MEDIUM issues (consider fixing)
